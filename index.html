Server Setup Guide

Setup Ubuntu servers for running Node.js, Angular and Docker

Initial System Setup

Update and Upgrade Packages

bash
apt update && apt upgrade -y && apt autoremove -y

Install Essential Tools

Essential packages for server management and security:

  • sudo: For privilege escalation
  • curl/wget: For downloading scripts/files
  • net-tools: For netstat etc.
  • nano: Text editor
  • fail2ban: Bans IPs after failed logins (e.g., SSH brute-force protection)
  • unattended-upgrades: For automatic security updates
bash
apt install sudo curl wget net-tools nano fail2ban unattended-upgrades -y

Configure Automatic Security Updates

bash
dpkg-reconfigure -plow unattended-upgrades

Create a Non-Root User

Set a strong password (at least 24 characters, mixed case/symbols)

bash
adduser produser  # Set a strong password
usermod -aG sudo produser  # Add to sudo group

Switch to this user:

bash
su - produser

Set Up SSH for Secure Remote Access

Install and Enable SSH Server

bash
sudo apt install openssh-server -y
sudo systemctl start ssh
sudo systemctl enable ssh

Generate SSH Keys on Your PC

bash
ssh-keygen -t ed25519

Copy Public Key

Windows:

powershell
Get-Content $HOME\.ssh\id_ed25519.pub

Linux:

bash
cat ~/.ssh/id_ed25519.pub

Add Public Key to Server

bash
mkdir -p ~/.ssh 
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys

Harden SSH Configuration

Edit SSH config:

bash
sudo nano /etc/ssh/sshd_config

Set these parameters:

config
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes

Restart SSH:

bash
sudo systemctl restart ssh

Configure Fail2Ban

Start and Enable Fail2Ban

bash
sudo systemctl start fail2ban
sudo systemctl enable fail2ban

Configure Jails

Edit jail configuration:

bash
sudo nano /etc/fail2ban/jail.local

Add this configuration:

config
[DEFAULT]
bantime  = 1h
findtime = 10m
maxretry = 50
backend  = systemd
banaction = ufw

[sshd]
enabled = true
port    = ssh
logpath = %(sshd_log)s

Restart Fail2Ban:

bash
sudo systemctl restart fail2ban

Verify jails:

bash
sudo fail2ban-client status

Configure Firewall (UFW)

Install UFW

bash
sudo apt install ufw -y

Set Default Policies

bash
sudo ufw default deny incoming
sudo ufw default allow outgoing

Secure SSH Access

Option 1: Allow from specific IP:

bash
sudo ufw allow from <YOUR_PUBLIC_IP> to any port 22 proto tcp

Option 2: Rate-limit SSH:

bash
sudo ufw limit ssh

Allow Web Traffic

bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

Enable UFW

bash
sudo ufw enable
sudo ufw status verbose

Install and Secure Web Server

Apache

Install Apache:

bash
sudo apt install apache2 -y

Start and Enable:

bash
sudo systemctl start apache2
sudo systemctl enable apache2

Harden Apache Config:

bash
sudo nano /etc/apache2/apache2.conf
apache
ServerTokens Prod
ServerSignature Off

<Directory /var/www/>
    Options -Indexes
    AllowOverride None
    Require all granted
</Directory>

Install ModSecurity:

bash
sudo apt install libapache2-mod-security2 -y
sudo a2enmod security2
sudo systemctl restart apache2

Enable ModSecurity:

bash
sudo nano /etc/modsecurity/modsecurity.conf

Set SecRuleEngine On

Install OWASP Core Rule Set:

bash
sudo apt install modsecurity-crs -y

Enable OWASP CRS:

bash
sudo cp /usr/share/modsecurity-crs/crs-setup.conf.example \
/etc/modsecurity/crs-setup.conf

OR

sudo cp /usr/share/modsecurity-crs/crs-setup.conf.example \
/etc/modsecurity/crs-setup.conf

Configure ModSecurity in Apache:

bash
sudo nano /etc/apache2/mods-enabled/security2.conf

Ensure it contains:

config
IncludeOptional /etc/modsecurity/*.conf
IncludeOptional /etc/modsecurity/crs-setup.conf
IncludeOptional /usr/share/modsecurity-crs/rules/*.conf

Test and Restart:

bash
apachectl configtest
sudo systemctl restart apache2

Angular SPA:

bash
sudo nano /etc/apache2/sites-available/app1.conf
apache
<VirtualHost *:80>
    ServerName app1.example.com
    DocumentRoot /var/www/angular-app1

    <Directory /var/www/angular-app1>
        AllowOverride All
        Require all granted
    </Directory>

    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.html [L]
</VirtualHost>

enable:

bash
sudo a2ensite app1

Node.js API (Reverse Proxy):

bash
sudo nano /etc/apache2/sites-available/api1.conf
apache
<VirtualHost *:80>
    ServerName api1.example.com

    ProxyPreserveHost On
    ProxyPass / http://127.0.0.1:3000/
    ProxyPassReverse / http://127.0.0.1:3000/

    RequestHeader set X-Forwarded-Proto "http"
</VirtualHost>

enable:

bash
sudo a2ensite api1
sudo systemctl reload apache2

Enable HTTPS with Certbot

bash
sudo apt install certbot python3-certbot-apache -y
sudo certbot --apache
sudo certbot renew --dry-run

Restart Apache:

bash
sudo systemctl restart apache2

Security Headers

bash
sudo nano /etc/apache2/conf-available/security-headers.conf
apache
Header always set X-Content-Type-Options "nosniff"
Header always set X-Frame-Options "DENY"
Header always set Referrer-Policy "strict-origin-when-cross-origin"
Header always set Strict-Transport-Security "max-age=63072000; includeSubDomains; preload"

enable:

bash
sudo a2enconf security-headers

Nginx

Install Nginx:

bash
sudo apt install nginx -y

Start and Enable:

bash
sudo systemctl start nginx
sudo systemctl enable nginx

Harden Nginx Config:

bash
sudo nano /etc/nginx/nginx.conf

Set these inside the http {} block.

nginx
server_tokens off;

# Performance & DoS protection
sendfile on;
tcp_nopush on;
tcp_nodelay on;
keepalive_timeout 65;
types_hash_max_size 2048;

# Prevent large request abuse
client_body_timeout 10s;
client_header_timeout 10s;
send_timeout 10s;

# Rate-limit zone for APIs
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;

Configure site (reverse proxy + headers):

bash
sudo nano /etc/nginx/sites-available/default

Edit default site:

nginx
server {
    listen 80;
    server_name example.com;

    # --------------------
    # Global security
    # --------------------
    server_tokens off;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # Disable directory listing everywhere by default
    autoindex off;

    
    # --------------------
    # Static website
    # --------------------
    location / {
        root /var/www/site;
        index index.html;
        try_files $uri $uri/ =404;
    }

    # --------------------
    # Node.js API (secured)
    # --------------------
    location /api/ {
        # Rate limit API
        # limit_req zone=api_limit burst=20 nodelay;

        # Block common bad methods
        limit_except GET POST PUT DELETE {
            deny all;
        }

        # Proxy settings
        proxy_pass http://127.0.0.1:3000/;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts (prevent slowloris)
        proxy_connect_timeout 5s;
        proxy_send_timeout 30s;
        proxy_read_timeout 30s;
    }

    # --------------------
    # File downloads (locked down)
    # --------------------
    location /files/ {
        alias /srv/files/;

        # Only allow GET
        limit_except GET {
            deny all;
        }

        # Prevent execution
        types {
            text/plain txt;
            application/pdf pdf;
        }

        default_type application/octet-stream;
    }

    # --------------------
    # Block hidden files
    # --------------------
    location ~ /\.(?!well-known) {
        deny all;
    }

    # --------------------
    # Size limits
    # --------------------
    client_max_body_size 10M;
}

Restart Nginx:

bash
sudo systemctl reload nginx

Install MongoDB

Import MongoDB GPG key:

bash
curl -fsSL https://pgp.mongodb.com/server-7.0.asc | \
sudo gpg --dearmor -o /usr/share/keyrings/mongodb-server.gpg

Add MongoDB repository:

bash
echo "deb [signed-by=/usr/share/keyrings/mongodb-server.gpg] \
https://repo.mongodb.org/apt/ubuntu $(lsb_release -cs)/mongodb-org/7.0 multiverse" | \
sudo tee /etc/apt/sources.list.d/mongodb-org.list

Install MongoDB:

bash
sudo apt update
sudo apt install -y mongodb-org

Start and enable MongoDB:

bash
sudo systemctl start mongod
sudo systemctl enable mongod

Change MongoDB Default Port

Edit MongoDB config:

bash
sudo nano /etc/mongod.conf

Change port and bind IP:

yaml
net:
  port: 42058
  bindIp: 127.0.0.1

Restart MongoDB:

bash
sudo systemctl restart mongod

Verify:

bash
ss -tulnp | grep mongod

Create Admin User

Connect locally:

bash
mongosh --port 42058

Create admin user:

js
use admin
db.createUser({
  user: "adminUser",
  pwd: "STRONG_PASSWORD_HERE",
  roles: [ { role: "userAdminAnyDatabase", db: "admin" } ]
})

Enable Authentication

Edit config:

bash
sudo nano /etc/mongod.conf

Enable auth:

yaml
security:
  authorization: enabled

Restart MongoDB:

bash
sudo systemctl restart mongod

Test login:

bash
mongosh --port 42058 -u adminUser -p --authenticationDatabase admin

Create App-Specific User

Create app user:

js
use myappdb
db.createUser({
  user: "appUser",
  pwd: "STRONG_PASSWORD_HERE",
  roles: [
    { role: "readWrite", db: "myappdb" }
  ]
})

Firewall Configuration

If MongoDB is local only:

bash
sudo ufw deny 42058

If accessed from another server (private IP only):

bash
sudo ufw allow from 10.0.0.5 to any port 42058

Check:

bash
sudo ufw status verbose

yaml

bash

bash
sudo systemctl restart mongod

Protect Against Brute Force (Fail2Ban)

Open mongodb config:

bash
sudo nano /etc/fail2ban/filter.d/mongodb.conf

Create filter:

bash
[Definition]
failregex = .*authentication failed.*client: <HOST>.*

Create jail:

bash
sudo nano /etc/fail2ban/jail.local
[mongodb]
enabled = true
port = 27018
filter = mongodb
logpath = /var/log/mongodb/mongod.log
maxretry = 15

Restart:

bash
sudo systemctl restart fail2ban
